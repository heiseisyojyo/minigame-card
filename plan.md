# 网页小游戏企划：牌店买取狂人

## 1. 项目概述（对外可见）
- **平台**：网页端，适配桌面与手机。
- **时长**：单局 10–15 分钟。
- **风格**：像素风、生活可爱、现代复古，参考开罗游戏；色彩明快但不高饱和。
- **主题**：牌店模拟经营（买取/回收卡牌）。
- **核心玩法**：验真 → 评品相 → 查价 → 定价 → 结算。

## 2. 主要画面与流程（对外可见）
### 2.1 标题画面
- 标题文案：**“牌店买取狂人”**
- 按钮：**开始游戏**

### 2.2 游戏主流程（7 天）
- 前 6 天游玩为手动买取流程。
- 第 7 天引入 “JHS APP” 一键拍照查卡与上架。

**每天流程（前 6 天）：**
1. **开始新一天**。
2. 从桌子外侧递来买取小筐（含多张卡）。
3. 点击开始进入**查验阶段**：
   - 点击卡片翻正反面，查看背面/边缘伤痕。
   - 桌上有**品相范例卡**（A/B/C）与**假卡范例**。
4. 判定真假与品相等级。
5. 打开桌面电脑查价：
   - X 系列 → X 系列网站
   - Y 系列 → Y 系列网站
6. 拖动卡片信息查看市场价，**给出定价**。
7. 完成一张 → 处理下一张。
8. 全部完成后**结算**：与系统预估差距越小，收益越高。
9. 进入下一天。

**第 7 天流程：**
1. 电脑提示新工具：**“JHS APP 上线”**。
2. 引导点击下载安装。
3. 买取需求到来，切换手机界面：
   - **拍照查卡**（按钮咔嚓）→ 直接显示卡名+价格。
   - 价格记录在列表底部。
4. 一键上架完成当天任务。

## 3. 难度与内容进阶（对外可见）
- **Day1**：5 张卡，全部 A 品相，全部 X 系列。
- **Day2**：引入 B 品相。
- **Day3**：引入 Y 系列。
- **Day4**：引入 C 品相。
- **Day5**：引入假卡。
- **Day6**：卡片数量增加，出现 X/Y 都搜不到的 **Z 系列**（需放弃）。
- **Day7**：JHS APP 一键处理，体现“效率大杀器”。

## 4. 卡牌规则与示例（对外可见）
### 4.1 品相标准
- **A 品**：无任何问题。
- **B 品**：轻微边缘发白或正面小伤痕。
- **C 品**：明显折痕或边缘损伤。
- **假卡**：卡背颜色不对、正面色调偏暗或偏淡。

### 4.2 卡牌系列风格
- **X 系列**：风格类似宝可梦卡牌。
- **Y 系列**：风格类似游戏王。
- **Z 系列**：风格类似万智牌（用于无法查询的特殊情况）。

## 5. 美术与 UI 方向（对外可见）
- **整体氛围**：像素风桌面经营，亲切可爱。
- **视角**：俯视桌面，玩家在桌前操作。
- **交互重点**：
  - 卡片翻面反馈（边缘提示）。
  - 电脑查价 UI（像素浏览器风格）。
  - 手机界面（像素 APP 风格）。

## 6. 音效与反馈（对外可见）
- 卡片翻面：轻微纸张声。
- 价格确认：像素提示音。
- 拍照：清脆快门声。

# 内部设计备注（不对外公开）
> 以下内容用于制作团队内部协作，不对玩家展示。

## A. 关卡节奏与教学设计
- Day1–2 以“认知规则”为主；Day3–4 加入系列/品相复杂度；Day5–6 混入假卡/无法查价卡。
- Day6 强制引导“尝试→查不到→放弃”的流程，降低玩家对全收购的执念。
- Day7 通过“一键上架”的爽感，形成明显反差。

## B. 经济与评分逻辑（内部实现）
- 评分基于“玩家定价与系统预估价”的误差区间：
  - 误差 ≤ 5%：满收益。
  - 误差 6–15%：收益递减。
  - 误差 > 15%：收益大幅降低。
- 假卡判定错误时，额外扣分或产生负收益。

## C. UI 交互节奏
- 每天开场 1–2 句提示，避免过多文本。
- 每张卡处理时 UI 仅保留“翻卡—查价—定价”三步，减少干扰。

## D. 文案规范
- 文案风格应更生活化，偏可爱、简短。
- 避免现代化术语堆叠，保持开罗游戏般轻量感。
